# Qwen 批量处理功能说明

## 概述
本项目实现了对Qwen大模型的批量处理支持，能够更高效地处理股票图表分析任务。

## 当前实现
当前实现提供了两种处理模式：

1. **批量模式**（默认）：收集多个请求然后并行处理，提高处理效率
2. **普通模式**：逐个发送API请求进行处理

## 批量处理功能特点

### 1. 请求收集与并行处理机制
- 系统会先收集指定数量的待处理请求（由`batch_size`参数控制）
- 当达到预设数量时，使用线程池并行处理这些请求
- 每张图像都会被独立分析和打分，确保与非批量模式结果一致

### 2. 配置参数
- `batch_mode`: 是否启用批量处理模式（布尔值），默认为true
- `batch_size`: 批量处理大小，默认为10个请求
- `batch_delay`: 批量处理间隔时间（秒）

### 3. 兼容性
- 与现有配置文件完全兼容
- 支持命令行参数覆盖配置
- 同时支持阿里云API和Ollama本地部署

## 使用方法

### 启用/禁用批量模式
```bash
# 默认情况下，批量模式已启用
python agent/qwen_review.py

# 显式启用批量模式（默认行为）
python agent/qwen_review.py --batch-mode

# 禁用批量模式，使用普通模式
python agent/qwen_review.py --no-batch-mode

# 指定批量大小
python agent/qwen_review.py --batch-size 20

# 通过配置文件控制
# 在 config/qwen_review.yaml 中设置:
# batch_mode: true  # 启用批量处理（默认）
# batch_mode: false # 禁用批量处理，使用普通模式
```

### 配置文件示例
```yaml
# config/qwen_review.yaml
batch_mode: true      # 启用批量处理（默认）
batch_size: 10        # 每批处理10个请求
batch_delay: 60       # 批量处理间隔60秒
```

## 技术细节

### 实现原理
- 当前的"批量处理"通过请求收集和线程池并行处理实现
- 每张图像仍然会单独发送到AI进行分析，确保分析质量和结果一致性
- 使用线程池实现真正的并行处理，而不是逐一处理
- 保留原有的限流和并发控制机制

### 限流控制
- 保留原有的并发控制机制
- 批量处理模式下同样应用RPM/TPM限流保护
- 支持多种限流配置档位（保守/标准/激进）

## 优势
1. **效率提升**：通过并行处理显著提高处理速度
2. **资源管理**：更好的内存和连接管理
3. **结果一致性**：每张图像都获得独立的分析和打分
4. **向后兼容**：不影响现有功能

## 注意事项
1. 批量处理模式现在是默认模式
2. 批量处理模式使用线程池并行处理请求
3. 每张图像都会独立分析，确保与非批量模式结果一致
4. 对于需要即时反馈的场景，可以使用普通模式（--no-batch-mode）
5. 批量大小应根据API限流策略和处理需求合理设置
